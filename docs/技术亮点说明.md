## 2. 核心技术亮点

### 2.1 Mesh 拓扑多人会议

**技术难点**：在无媒体服务器的情况下实现多人实时音视频通信。

```
传统方案（SFU/MCU）vs MiniMeeting 方案（Mesh）

传统 SFU 方案：
┌─────┐         ┌─────┐
│用户A│────────►│     │
└─────┘         │ SFU │         ┌─────┐
                │     │────────►│用户C│
┌─────┐         │     │         └─────┘
│用户B│────────►│     │
└─────┘         └─────┘
优点：客户端压力小，支持大规模
缺点：需要部署 SFU 服务器

MiniMeeting Mesh 方案：
        ┌─────┐
        │用户A│
        └──┬──┘
       ╱   │   ╲
      ╱    │    ╲
┌───┴──┐ ┌─┴──┐ ┌───┴──┐
│用户B │ │用户C│ │用户D │
└──────┘ └────┘ └──────┘

优点：无需媒体服务器，部署简单
缺点：客户端压力大，规模受限

创新点：
1. 动态带宽分配：根据发言者优先级调整带宽
2. 自适应分辨率：按需降级非发言者视频质量
3. 智能布局：根据参与者数量动态调整窗口布局
```

**实现细节**：

```cpp
// Mesh 连接管理
class MeshConnectionManager {
public:
    // 动态调整各路视频质量
    void adjustQualityByPriority(const QString& speakerId) {
        for (auto& participant : m_participants) {
            if (participant.id() == speakerId) {
                // 发言者：高画质
                participant.setVideoQuality(VideoQuality::High);
                participant.setBitrate(500);  // kbps
                participant.setResolution(1280, 720);
            } else {
                // 非发言者：低画质
                participant.setVideoQuality(VideoQuality::Low);
                participant.setBitrate(100);  // kbps
                participant.setResolution(320, 180);
                participant.setFrameRate(5);  // 低帧率
            }
        }
    }
    
    // 带宽总控
    void enforceTotalBandwidthLimit() {
        int totalBitrate = 0;
        for (const auto& p : m_participants) {
            totalBitrate += p.bitrate();
        }
        
        // 超出总带宽限制时，按优先级降级
        while (totalBitrate > m_maxTotalBitrate) {
            auto lowest = findLowestPriorityParticipant();
            lowest->decreaseBitrate();
            totalBitrate -= 100;
        }
    }
};
```

**GCC 拥塞控制实现原理**：

```cpp
// GCC (Google Congestion Control) 核心算法
class GCCController {
public:
    void onPacketArrival(const RtpPacket& packet) {
        // 1. 计算延迟梯度
        double delay = packet.arrivalTime - packet.sendTime;
        double delayGradient = delay - m_lastDelay;
        
        // 2. 趋势检测
        m_trendLine.update(delayGradient);
        double trend = m_trendLine.getSlope();
        
        // 3. 判断拥塞状态
        if (trend > m_threshold) {
            // 网络拥塞：降低发送速率
            m_targetBitrate *= 0.85;
        } else if (trend < m_threshold * 0.5) {
            // 网络空闲：增加发送速率
            m_targetBitrate *= 1.03;
        }
        
        // 4. 限制范围
        m_targetBitrate = std::max(m_minBitrate, 
                                   std::min(m_maxBitrate, m_targetBitrate));
        
        // 5. 通知编码器
        m_encoder->setBitrate(m_targetBitrate);
    }
    
private:
    double m_targetBitrate = 300;  // kbps
    double m_minBitrate = 50;
    double m_maxBitrate = 1500;
    double m_threshold = 0.05;
    TrendLine m_trendLine;
};
```

### 2.4 Qt 与 WebRTC 集成

**技术难点**：将 C++ 原生 WebRTC 与 Qt 框架深度集成。

```
集成挑战：

1. 线程模型差异
   - WebRTC：多线程，专用工作线程
   - Qt：主事件循环，UI 线程

2. 回调机制差异
   - WebRTC：C++ 虚函数回调
   - Qt：信号槽机制

3. 资源管理差异
   - WebRTC：引用计数 (rtc::scoped_refptr)
   - Qt：父子对象树 (QObject)

解决方案：

┌────────────────────────────────────────────────────────────────────┐
│                     Qt-WebRTC 桥接层                               │
├────────────────────────────────────────────────────────────────────┤
│                                                                    │
│  Qt 应用层                                                         │
│  ┌──────────────────────────────────────────────────────────────┐ │
│  │  QObjects ──► Signals/Slots ──► UI Updates                   │ │
│  └──────────────────────────────────────────────────────────────┘ │
│         │                                                          │
│         │ 信号槽连接（自动线程安全）                               │
│         ▼                                                          │
│  桥接层                                                            │
│  ┌──────────────────────────────────────────────────────────────┐ │
│  │  Observer 类                                                 │ │
│  │  - 继承 WebRTC Observer 接口                                 │ │
│  │  - 在回调中发射 Qt 信号                                      │ │
│  │  - 自动处理线程切换                                          │ │
│  └──────────────────────────────────────────────────────────────┘ │
│         │                                                          │
│         │ 回调                                                     │
│         ▼                                                          │
│  WebRTC 层                                                         │
│  ┌──────────────────────────────────────────────────────────────┐ │
│  │  PeerConnection, MediaStream, etc.                          │ │
│  │  工作线程执行                                                │ │
│  └──────────────────────────────────────────────────────────────┘ │
│                                                                    │
└────────────────────────────────────────────────────────────────────┘
```

**桥接实现**：

```cpp
// WebRTC Observer 到 Qt 信号的桥接
class PeerConnectionObserverBridge 
    : public QObject
    , public webrtc::PeerConnectionObserver
{
    Q_OBJECT
    
public:
    // WebRTC 回调（工作线程）
    void OnIceCandidate(const webrtc::IceCandidateInterface* candidate) override {
        std::string sdp;
        candidate->ToString(&sdp);
        
        // 发射 Qt 信号（自动切换到 UI 线程）
        emit iceCandidateReady(
            QString::fromStdString(sdp),
            QString::fromStdString(candidate->sdp_mid()),
            candidate->sdp_mline_index()
        );
    }
    
    void OnTrack(rtc::scoped_refptr<webrtc::MediaStreamTrackInterface> track) override {
        if (track->kind() == webrtc::MediaStreamTrackInterface::kVideoKind) {
            auto* videoTrack = static_cast<webrtc::VideoTrackInterface*>(track.get());
            emit remoteVideoTrackAdded(new QtVideoTrack(videoTrack, this));
        }
    }
    
    void OnIceConnectionChange(webrtc::PeerConnectionInterface::IceConnectionState state) override {
        emit iceConnectionStateChanged(static_cast<int>(state));
    }
    
signals:
    void iceCandidateReady(const QString& sdp, const QString& sdpMid, int sdpMlineIndex);
    void remoteVideoTrackAdded(QtVideoTrack* track);
    void iceConnectionStateChanged(int state);
};

// 使用
auto bridge = new PeerConnectionObserverBridge(this);
connect(bridge, &PeerConnectionObserverBridge::iceCandidateReady,
        this, &MeetingSession::sendIceCandidate);
connect(bridge, &PeerConnectionObserverBridge::remoteVideoTrackAdded,
        this, &MeetingSession::addRemoteVideo);
```

## 5. 面试常见问题与回答

### Q1: 为什么选择 Mesh 拓扑而不是 SFU？

**回答要点**：

```
1. 产品定位：
   - 目标是技术面试展示和小规模会议
   - 2-6 人规模下 Mesh 完全够用

2. 成本优势：
   - 无需部署媒体服务器
   - 运维成本几乎为零
   - 适合个人开发者

3. 学习价值：
   - 完整展示 P2P 连接过程
   - 深入理解 WebRTC 核心机制

4. 性能可接受：
   - 6 人会议总带宽约 3Mbps
   - 现代 CPU 完全可以承受

5. 扩展性考虑：
   - 如需扩展，可平滑升级到 SFU
   - 信令协议无需改变
```

### Q2: 如何处理对称 NAT 的穿透问题？

**回答要点**：

```
1. 问题分析：
   - 对称 NAT 对每个目标地址创建不同映射
   - 导致 STUN 获取的地址无法被对方使用

2. 解决方案：
   - 部署 TURN 服务器作为中继
   - 支持 TURN over TLS 提高安全性
   - 动态凭证生成防止滥用

3. 优化策略：
   - 优先尝试 P2P 穿透
   - 仅在穿透失败时使用 TURN
   - 根据网络状况自动切换

4. 实现细节：
   - 配置多个 TURN 服务器提高可用性
   - 支持 TCP/TLS 作为 UDP 失败的备选
```

### Q3: 如何优化弱网环境下的通话质量？

**回答要点**：

```
1. 网络质量检测：
   - 实时监控丢包率、RTT、抖动
   - 使用 WebRTC getStats() API 获取统计

2. 多层降级策略：
   - 第一层：GCC 自动调节码率
   - 第二层：FEC 前向纠错
   - 第三层：NACK 重传
   - 第四层：降低分辨率和帧率

3. 音频优先：
   - 网络拥塞时优先保证音频
   - 使用 Opus 编解码器
   - 启用带内 FEC

4. 自适应策略：
   - 根据网络状况动态调整
   - 平滑过渡避免突变
```

### Q4: 如何实现端到端加密？

**回答要点**：

```
1. DTLS 握手：
   - 建立连接时进行 DTLS 握手
   - 双方交换证书指纹
   - 通过 SDP 中的 fingerprint 验证

2. SRTP 加密：
   - DTLS 握手后导出密钥
   - 使用 AES-128-CTR 加密媒体
   - HMAC-SHA1 保证完整性

3. 前向保密：
   - 使用 ECDHE 密钥交换
   - 每个会话独立密钥
   - 即使泄露一个密钥也不影响其他会话

4. 信令安全：
   - WebSocket over TLS (WSS)
   - 防止中间人攻击
```

### Q5: 项目中遇到的最大挑战是什么？

**回答要点**：

```
挑战：Qt 与 WebRTC 的跨线程集成

问题描述：
- WebRTC 回调在工作线程
- Qt UI 操作必须在主线程
- 直接操作会导致崩溃

解决方案：
1. 设计桥接层模式
   - Observer 类同时继承 QObject 和 WebRTC Observer
   - 在回调中发射 Qt 信号

2. 利用 Qt 信号槽的线程安全特性
   - 跨线程信号自动队列化
   - 在接收者所在线程执行

3. 资源管理统一
   - 使用智能指针管理 WebRTC 对象
   - 使用 Qt 父子关系管理 UI 对象

收获：
- 深入理解多线程编程
- 掌握跨框架集成技巧
- 提高系统设计能力
```
