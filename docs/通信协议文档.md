# MiniMeeting 通信协议文档

## 1. 协议概述

MiniMeeting 使用 WebSocket 作为信令通道的传输层协议，所有信令消息均采用 JSON 格式进行序列化。

### 1.1 通信架构

```
┌──────────────┐                    ┌──────────────┐
│   Client A   │◄───WebSocket──────►│              │
└──────────────┘                    │   Signaling  │
                                    │   Server     │
┌──────────────┐                    │              │
│   Client B   │◄───WebSocket──────►│              │
└──────────────┘                    └──────────────┘
```

### 1.2 消息格式

所有消息均为 JSON 格式，基础结构如下：

```json
{
    "type": "<消息类型>",
    // ... 其他字段
}
```

## 2. 客户端 → 服务器消息

### 2.1 登录消息 (login)

用户连接到信令服务器后，发送登录消息进行身份注册。

**请求格式：**
```json
{
    "type": "login",
    "userId": "user_001",
    "userName": "张三"
}
```

**字段说明：**

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| type | string | 是 | 固定值 "login" |
| userId | string | 是 | 用户唯一标识，客户端生成或服务端分配 |
| userName | string | 否 | 用户显示名称 |

### 2.2 创建房间消息 (createRoom)

用户创建一个新的会议室。

**请求格式：**
```json
{
    "type": "createRoom",
    "userId": "user_001",
    "roomName": "项目讨论会"
}
```

**字段说明：**

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| type | string | 是 | 固定值 "createRoom" |
| userId | string | 是 | 创建者用户ID |
| roomName | string | 否 | 房间名称，默认为 "会议室" |

### 2.3 加入房间消息 (joinRoom)

用户加入指定的会议室。

**请求格式：**
```json
{
    "type": "joinRoom",
    "userId": "user_002",
    "roomId": "room_001"
}
```

**字段说明：**

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| type | string | 是 | 固定值 "joinRoom" |
| userId | string | 是 | 加入者用户ID |
| roomId | string | 是 | 目标房间ID |

### 2.4 离开房间消息 (leaveRoom)

用户离开当前所在的房间。

**请求格式：**
```json
{
    "type": "leaveRoom",
    "userId": "user_001",
    "roomId": "room_001"
}
```

**字段说明：**

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| type | string | 是 | 固定值 "leaveRoom" |
| userId | string | 是 | 离开者用户ID |
| roomId | string | 是 | 房间ID |

### 2.5 获取房间列表消息 (getRoomList)

获取当前所有可用的房间列表。

**请求格式：**
```json
{
    "type": "getRoomList",
    "userId": "user_001"
}
```

**字段说明：**

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| type | string | 是 | 固定值 "getRoomList" |
| userId | string | 是 | 请求者用户ID |

### 2.6 Offer SDP 消息 (offer)

发送 WebRTC Offer SDP 给对端用户。

**请求格式：**
```json
{
    "type": "offer",
    "from": "user_001",
    "to": "user_002",
    "roomId": "room_001",
    "sdp": "v=0\r\no=- 123456789 2 IN IP4 127.0.0.1\r\n..."
}
```

**字段说明：**

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| type | string | 是 | 固定值 "offer" |
| from | string | 是 | 发送者用户ID |
| to | string | 是 | 接收者用户ID |
| roomId | string | 是 | 房间ID |
| sdp | string | 是 | SDP Offer 内容 |

### 2.7 Answer SDP 消息 (answer)

发送 WebRTC Answer SDP 给对端用户。

**请求格式：**
```json
{
    "type": "answer",
    "from": "user_002",
    "to": "user_001",
    "roomId": "room_001",
    "sdp": "v=0\r\no=- 987654321 2 IN IP4 127.0.0.1\r\n..."
}
```

**字段说明：**

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| type | string | 是 | 固定值 "answer" |
| from | string | 是 | 发送者用户ID |
| to | string | 是 | 接收者用户ID |
| roomId | string | 是 | 房间ID |
| sdp | string | 是 | SDP Answer 内容 |

### 2.8 ICE Candidate 消息 (iceCandidate)

发送 ICE Candidate 给对端用户。

**请求格式：**
```json
{
    "type": "iceCandidate",
    "from": "user_001",
    "to": "user_002",
    "roomId": "room_001",
    "candidate": {
        "candidate": "candidate:1 1 UDP 2122260223 192.168.1.100 54321 typ host",
        "sdpMid": "0",
        "sdpMLineIndex": 0
    }
}
```

**字段说明：**

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| type | string | 是 | 固定值 "iceCandidate" |
| from | string | 是 | 发送者用户ID |
| to | string | 是 | 接收者用户ID |
| roomId | string | 是 | 房间ID |
| candidate | object | 是 | ICE Candidate 对象 |
| candidate.candidate | string | 是 | Candidate 字符串 |
| candidate.sdpMid | string | 是 | SDP Media ID |
| candidate.sdpMLineIndex | number | 是 | SDP Media Line Index |

### 2.9 心跳消息 (heartbeat)

客户端定期发送心跳以保持连接活跃。

**请求格式：**
```json
{
    "type": "heartbeat",
    "userId": "user_001",
    "timestamp": 1700000000000
}
```

**字段说明：**

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| type | string | 是 | 固定值 "heartbeat" |
| userId | string | 是 | 用户ID |
| timestamp | number | 否 | 客户端时间戳（毫秒） |

## 3. 服务器 → 客户端消息

### 3.1 登录成功消息 (loginSuccess)

登录成功后返回。

**响应格式：**
```json
{
    "type": "loginSuccess",
    "userId": "user_001"
}
```

### 3.2 登录失败消息 (loginFailed)

登录失败时返回。

**响应格式：**
```json
{
    "type": "loginFailed",
    "errorCode": 1001,
    "message": "用户名已存在"
}
```

### 3.3 房间创建成功消息 (roomCreated)

房间创建成功后返回。

**响应格式：**
```json
{
    "type": "roomCreated",
    "roomId": "room_001",
    "roomName": "项目讨论会",
    "creatorId": "user_001"
}
```

### 3.4 加入房间成功消息 (joinRoomSuccess)

成功加入房间后返回。

**响应格式：**
```json
{
    "type": "joinRoomSuccess",
    "roomId": "room_001",
    "roomName": "项目讨论会",
    "users": ["user_001", "user_002"]
}
```

### 3.5 房间列表消息 (roomList)

返回当前可用房间列表。

**响应格式：**
```json
{
    "type": "roomList",
    "rooms": [
        {
            "roomId": "room_001",
            "roomName": "项目讨论会",
            "userCount": 2,
            "creatorId": "user_001"
        },
        {
            "roomId": "room_002",
            "roomName": "周会",
            "userCount": 1,
            "creatorId": "user_003"
        }
    ]
}
```

### 3.6 用户加入通知 (userJoined)

有新用户加入房间时广播给房间内其他用户。

**响应格式：**
```json
{
    "type": "userJoined",
    "roomId": "room_001",
    "userId": "user_002",
    "userName": "李四"
}
```

### 3.7 用户离开通知 (userLeft)

有用户离开房间时广播给房间内其他用户。

**响应格式：**
```json
{
    "type": "userLeft",
    "roomId": "room_001",
    "userId": "user_001"
}
```

### 3.8 收到 Offer 消息 (offer)

服务器转发 Offer SDP 给目标用户。

**响应格式：**
```json
{
    "type": "offer",
    "from": "user_001",
    "roomId": "room_001",
    "sdp": "v=0\r\no=- 123456789 2 IN IP4 127.0.0.1\r\n..."
}
```

### 3.9 收到 Answer 消息 (answer)

服务器转发 Answer SDP 给目标用户。

**响应格式：**
```json
{
    "type": "answer",
    "from": "user_002",
    "roomId": "room_001",
    "sdp": "v=0\r\no=- 987654321 2 IN IP4 127.0.0.1\r\n..."
}
```

### 3.10 收到 ICE Candidate 消息 (iceCandidate)

服务器转发 ICE Candidate 给目标用户。

**响应格式：**
```json
{
    "type": "iceCandidate",
    "from": "user_001",
    "roomId": "room_001",
    "candidate": {
        "candidate": "candidate:1 1 UDP 2122260223 192.168.1.100 54321 typ host",
        "sdpMid": "0",
        "sdpMLineIndex": 0
    }
}
```

### 3.11 心跳响应 (heartbeatAck)

服务器响应心跳消息。

**响应格式：**
```json
{
    "type": "heartbeatAck",
    "timestamp": 1700000000000
}
```

### 3.12 错误消息 (error)

发生错误时返回。

**响应格式：**
```json
{
    "type": "error",
    "errorCode": 1002,
    "message": "房间不存在"
}
```

## 4. 消息流程示例

### 4.1 完整通话建立流程

```
Client A                    Server                    Client B
    │                         │                          │
    │──── login ─────────────►│                          │
    │◄─── loginSuccess ───────│                          │
    │                         │                          │
    │──── createRoom ────────►│                          │
    │◄─── roomCreated ────────│                          │
    │                         │                          │
    │                         │◄──── login ──────────────│
    │                         │───── loginSuccess ──────►│
    │                         │                          │
    │                         │◄──── joinRoom ───────────│
    │◄─── userJoined ─────────│───── joinRoomSuccess ──►│
    │                         │                          │
    │──── offer ─────────────►│                          │
    │                         │───── offer ─────────────►│
    │                         │                          │
    │                         │◄──── answer ─────────────│
    │◄─── answer ─────────────│                          │
    │                         │                          │
    │──── iceCandidate ──────►│                          │
    │                         │───── iceCandidate ──────►│
    │                         │                          │
    │                         │◄──── iceCandidate ───────│
    │◄─── iceCandidate ───────│                          │
    │                         │                          │
    ══════════════ WebRTC P2P 连接建立 ══════════════════
    │                                                    │
```

### 4.2 心跳保活流程

```
Client                      Server
    │                         │
    │──── heartbeat ─────────►│
    │◄─── heartbeatAck ───────│
    │                         │
    │   (等待 30 秒)           │
    │                         │
    │──── heartbeat ─────────►│
    │◄─── heartbeatAck ───────│
    │                         │
```

## 5. 心跳机制

### 5.1 心跳参数

| 参数 | 默认值 | 说明 |
|------|--------|------|
| 心跳间隔 | 30 秒 | 客户端发送心跳的间隔 |
| 超时时间 | 90 秒 | 服务器判定用户离线的超时时间 |
| 最大重试次数 | 3 次 | 心跳失败后的重试次数 |

### 5.2 心跳流程

1. 客户端每 30 秒发送一次 `heartbeat` 消息
2. 服务器收到后更新用户的 `lastHeartbeat` 时间戳，并返回 `heartbeatAck`
3. 若服务器在 90 秒内未收到某用户的心跳，则判定用户离线
4. 用户离线时，服务器清理该用户资源，并通知房间内其他用户

## 6. 错误处理

### 6.1 错误码定义

| 错误码 | 错误类型 | 说明 |
|--------|----------|------|
| 1001 | LOGIN_FAILED | 登录失败（用户名已存在等） |
| 1002 | ROOM_NOT_FOUND | 房间不存在 |
| 1003 | ROOM_FULL | 房间已满 |
| 1004 | NOT_LOGGED_IN | 未登录 |
| 1005 | ALREADY_IN_ROOM | 用户已在房间中 |
| 1006 | NOT_IN_ROOM | 用户不在该房间中 |
| 1007 | INVALID_MESSAGE | 无效的消息格式 |
| 1008 | TARGET_NOT_FOUND | 目标用户不存在 |
| 1009 | PERMISSION_DENIED | 权限不足 |
| 1010 | INTERNAL_ERROR | 服务器内部错误 |

### 6.2 错误响应格式

```json
{
    "type": "error",
    "errorCode": 1002,
    "message": "房间不存在",
    "requestType": "joinRoom"
}
```

## 7. 安全考虑

### 7.1 连接安全

- WebSocket 使用 WSS（WebSocket Secure）协议
- 服务器需要配置有效的 SSL/TLS 证书

### 7.2 消息验证

- 所有消息必须包含 `type` 字段
- 服务器需验证消息格式和必填字段
- 用户 ID 和房间 ID 需验证合法性

### 7.3 资源限制

| 限制项 | 限制值 |
|--------|--------|
| 单房间最大人数 | 4 人 |
| 用户名最大长度 | 32 字符 |
| 房间名最大长度 | 64 字符 |
| 单条消息最大长度 | 64 KB |
