# MiniMeeting 部署安装手册

## 1. 环境要求

### 1.1 客户端环境

| 组件 | 最低版本 | 推荐版本 |
|------|----------|----------|
| 操作系统 | Windows 10 | Windows 10/11 64位 |
| 编译器 | Visual Studio 2019 | Visual Studio 2022 |
| CMake | 3.20 | 3.25+ |
| Qt | 6.2 | 6.5+ |
| WebRTC | M120 | M120+ |

### 1.2 服务器环境

| 组件 | 最低版本 | 推荐版本 |
|------|----------|----------|
| 操作系统 | Ubuntu 18.04 | Ubuntu 22.04 LTS |
| Node.js | 16.x | 18.x LTS |
| npm | 8.x | 9.x+ |
| coturn | 4.5 | 4.6+ |

### 1.3 网络要求

| 端口 | 协议 | 用途 |
|------|------|------|
| 8080 | TCP | 信令服务器 WebSocket |
| 3478 | UDP/TCP | STUN/TURN 服务 |
| 5349 | TCP | TURN TLS |
| 49152-65535 | UDP | WebRTC 媒体传输 |

## 2. 客户端编译安装

### 2.1 准备工作

#### 2.1.1 安装 Visual Studio 2022

1. 下载 Visual Studio 2022 Community：https://visualstudio.microsoft.com/
2. 安装时选择以下组件：
   - 使用 C++ 的桌面开发
   - Windows 10/11 SDK
   - CMake 工具

#### 2.1.2 安装 CMake

```powershell
# 使用 winget 安装
winget install Kitware.CMake

# 或从官网下载安装
# https://cmake.org/download/
```

#### 2.1.3 安装 Qt

1. 下载 Qt Online Installer：https://www.qt.io/download
2. 安装 Qt 6.x，选择以下组件：
   - MSVC 2019/2022 64-bit
   - Qt Network Authorization
   - Qt WebSockets

#### 2.1.4 设置环境变量

```powershell
# 设置 Qt 路径（根据实际安装路径调整）
$env:Qt6_DIR = "C:\Qt\6.5.0\msvc2019_64\lib\cmake\Qt6"

# 添加到用户环境变量
[Environment]::SetEnvironmentVariable("Qt6_DIR", "C:\Qt\6.5.0\msvc2019_64\lib\cmake\Qt6", "User")
```

### 2.2 编译 WebRTC Native

#### 2.2.1 安装 depot_tools

```powershell
# 克隆 depot_tools
git clone https://chromium.googlesource.com/chromium/tools/depot_tools.git

# 添加到 PATH
$env:PATH += ";C:\path\to\depot_tools"
```

#### 2.2.2 下载 WebRTC 源码

```powershell
# 创建工作目录
mkdir webrtc-checkout
cd webrtc-checkout

# 下载源码（约 15GB，耗时较长）
fetch --nohooks webrtc
gclient sync
```

#### 2.2.3 编译 WebRTC

```powershell
cd src

# 生成编译配置
gn gen out/Debug --args="target_os=\"win\" target_cpu=\"x64\" is_debug=true rtc_include_tests=false rtc_build_examples=false"

# 编译（使用多核加速）
ninja -C out/Debug -j 8
```

#### 2.2.4 编译 Release 版本

```powershell
gn gen out/Release --args="target_os=\"win\" target_cpu=\"x64\" is_debug=false rtc_include_tests=false rtc_build_examples=false"
ninja -C out/Release -j 8
```

### 2.3 编译 MiniMeeting 客户端

#### 2.3.1 克隆项目

```powershell
git clone https://github.com/your-org/MiniMeeting.git
cd MiniMeeting
```

#### 2.3.2 配置 WebRTC 路径

编辑 `client/CMakeLists.txt`，设置 WebRTC 路径：

```cmake
set(WEBRTC_ROOT_DIR "C:/path/to/webrtc-checkout/src")
```

#### 2.3.3 编译项目

```powershell
cd client

# 创建编译目录
mkdir build
cd build

# 生成项目文件
cmake .. -G "Visual Studio 17 2022" -A x64

# 编译 Debug 版本
cmake --build . --config Debug

# 或编译 Release 版本
cmake --build . --config Release
```

#### 2.3.4 运行客户端

```powershell
# Debug 版本
.\Debug\MiniMeeting.exe

# Release 版本
.\Release\MiniMeeting.exe
```

## 3. 信令服务器部署

### 3.1 安装 Node.js

```bash
# Ubuntu/Debian
curl -fsSL https://deb.nodesource.com/setup_18.x | sudo -E bash -
sudo apt-get install -y nodejs

# 验证安装
node --version
npm --version
```

### 3.2 部署信令服务器

```bash
# 进入服务器目录
cd MiniMeeting/server

# 安装依赖
npm install

# 配置服务器（编辑配置文件）
vim config/default.json
```

**config/default.json 示例：**
```json
{
    "server": {
        "port": 8080,
        "host": "0.0.0.0"
    },
    "heartbeat": {
        "interval": 30000,
        "timeout": 90000
    },
    "room": {
        "maxUsers": 4
    },
    "ice": {
        "stunServers": [
            "stun:stun.l.google.com:19302",
            "stun:stun1.l.google.com:19302"
        ],
        "turnServer": "turn:your-server.com:3478",
        "turnUsername": "your-username",
        "turnCredential": "your-password"
    }
}
```

### 3.3 启动服务器

```bash
# 开发模式
npm run dev

# 生产模式
npm start

# 使用 PM2 管理进程
npm install -g pm2
pm2 start src/index.js --name minimeeting-signaling
pm2 save
pm2 startup
```

### 3.4 配置 Nginx 反向代理

```nginx
# /etc/nginx/sites-available/minimeeting
server {
    listen 80;
    server_name your-domain.com;

    location /ws {
        proxy_pass http://127.0.0.1:8080;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_read_timeout 86400;
    }
}
```

```bash
# 启用配置
sudo ln -s /etc/nginx/sites-available/minimeeting /etc/nginx/sites-enabled/
sudo nginx -t
sudo systemctl reload nginx
```

## 4. TURN 服务器部署

### 4.1 安装 coturn

```bash
# Ubuntu/Debian
sudo apt-get update
sudo apt-get install -y coturn

# CentOS/RHEL
sudo yum install -y coturn
```

### 4.2 配置 coturn

编辑 `/etc/turnserver.conf`：

```conf
# 监听端口
listening-port=3478
tls-listening-port=5349

# 监听 IP（设置为服务器公网 IP）
listening-ip=YOUR_SERVER_IP
relay-ip=YOUR_SERVER_IP

# 外网 IP（NAT 环境）
external-ip=YOUR_PUBLIC_IP

# Realm
realm=your-domain.com

# 认证
lt-cred-mech
user=username:password

# TLS 证书（可选，用于安全连接）
cert=/etc/ssl/certs/your-cert.pem
pkey=/etc/ssl/private/your-key.pem

# 日志
log-file=/var/log/turnserver.log
verbose

# 安全设置
no-multicast-peers
no-cli

# 中继端口范围
min-port=49152
max-port=65535
```

### 4.3 启动 coturn

```bash
# 启动服务
sudo systemctl enable coturn
sudo systemctl start coturn

# 检查状态
sudo systemctl status coturn

# 检查日志
sudo tail -f /var/log/turnserver.log
```

### 4.4 验证 TURN 服务

```bash
# 使用 turnutils 测试
turnutils_uclient -v -t -u username -w password YOUR_SERVER_IP
```

## 5. 防火墙配置

### 5.1 Ubuntu UFW

```bash
# 允许必要端口
sudo ufw allow 22/tcp       # SSH
sudo ufw allow 80/tcp       # HTTP
sudo ufw allow 443/tcp      # HTTPS
sudo ufw allow 8080/tcp     # 信令服务器
sudo ufw allow 3478/tcp     # STUN/TURN TCP
sudo ufw allow 3478/udp     # STUN/TURN UDP
sudo ufw allow 5349/tcp     # TURN TLS
sudo ufw allow 49152:65535/udp  # WebRTC 媒体端口

# 启用防火墙
sudo ufw enable
```

### 5.2 CentOS Firewalld

```bash
# 允许必要端口
sudo firewall-cmd --permanent --add-port=22/tcp
sudo firewall-cmd --permanent --add-port=80/tcp
sudo firewall-cmd --permanent --add-port=443/tcp
sudo firewall-cmd --permanent --add-port=8080/tcp
sudo firewall-cmd --permanent --add-port=3478/tcp
sudo firewall-cmd --permanent --add-port=3478/udp
sudo firewall-cmd --permanent --add-port=5349/tcp
sudo firewall-cmd --permanent --add-port=49152-65535/udp

# 重载配置
sudo firewall-cmd --reload
```

## 6. SSL/TLS 配置

### 6.1 使用 Let's Encrypt

```bash
# 安装 Certbot
sudo apt-get install -y certbot python3-certbot-nginx

# 获取证书
sudo certbot --nginx -d your-domain.com

# 自动续期
sudo certbot renew --dry-run
```

### 6.2 配置 WSS

修改 Nginx 配置支持 WebSocket Secure：

```nginx
server {
    listen 443 ssl;
    server_name your-domain.com;

    ssl_certificate /etc/letsencrypt/live/your-domain.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/your-domain.com/privkey.pem;

    location /ws {
        proxy_pass http://127.0.0.1:8080;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_read_timeout 86400;
    }
}
```

## 8. 故障排查

### 8.1 常见问题

| 问题 | 可能原因 | 解决方案 |
|------|----------|----------|
| 客户端无法连接信令服务器 | 防火墙阻止 | 检查端口开放状态 |
| WebRTC 连接失败 | NAT 穿透问题 | 检查 STUN/TURN 配置 |
| 音视频卡顿 | 带宽不足 | 检查网络质量，启用弱网降级 |
| TURN 中继不工作 | 认证失败 | 检查用户名密码配置 |

### 8.2 调试命令

```bash
# 检查端口监听
sudo netstat -tuln | grep 8080

# 测试 WebSocket 连接
wscat -c ws://your-server:8080

# 测试 TURN 服务
turnutils_uclient -v -u username -w password your-turn-server

# 查看 coturn 日志
sudo tail -f /var/log/turnserver.log
```
